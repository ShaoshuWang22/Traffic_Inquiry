# 交通出行规划系统 - 新版使用说明-名称：7.2我们拥有了35个城市牛逼(1)
## 项目概述
本项目是一款基于图形化界面的交通出行规划系统，采用邻接表存储的图结构实现全国多城市间的航班、火车出行路线规划。系统支持按**最短时间**或**最低成本**两种策略规划路线，可指定途经城市，覆盖华北、东北、华东、华中、华南、西南、西北等区域的数十个核心城市，为用户提供精准的跨城出行方案。

## 核心特性
1. **城市覆盖**：包含乌鲁木齐、西宁、兰州、银川、呼和浩特、北京、天津、石家庄、太原、哈尔滨、长春、沈阳、大连、西安、郑州、济南、青岛、合肥、南京、上海、杭州、武汉、南昌、长沙、成都、重庆、贵阳、昆明、广州、深圳、南宁、海口等30+核心城市；
2. **交通类型**：支持飞机（TYPE_FLIGHT）、火车（TYPE_TRAIN）两种交通方式，每种方式包含多个真实模拟班次；
3. **规划策略**：
   - 策略1：优先最短到达时间（含换乘等待时间，飞机换乘需≥120分钟，火车换乘需≥60分钟）；
   - 策略2：优先最低出行成本（成本相同时优先更短时间）；
4. **途经点支持**：可指定多个途经城市，系统自动排列组合途经顺序，生成最优完整路线；
5. **时间处理**：支持跨天时间计算，自动转换“时分”格式与分钟数，输出带天数的到达/出发时间；
6. **图形化交互**：基于EasyX图形库实现可视化地图，城市节点、交通线路直观展示。

## 数据结构设计
### 核心结构体
1. **Schedule（班次信息）**：存储班次号、出发时间（分钟）、到达时间（分钟）、费用、下一班次指针；
2. **ArcNode（边节点）**：存储目标城市下标、里程、交通类型、班次链表、下一条边指针；
3. **VNode（顶点节点）**：存储城市名称、三字代码、地图像素坐标、边链表头指针；
4. **Graph（图结构）**：存储城市顶点数组、顶点数量；
5. **PathState（路径状态）**：存储最优值（时间/成本）、前驱城市、选中班次、到达绝对时间；
6. **GlobalResult（全局结果）**：存储完整路径、选中班次、绝对出发/到达时间、总耗时、总成本。

## 核心功能函数
### 1. 基础工具函数
| 函数名 | 功能 |
|--------|------|
| `TimeToMin` | 将“HH:MM”格式时间转换为分钟数 |
| `MinToTimeStr` | 将分钟数转换为带天数的时间字符串（如“当天 08:30”“第2天 10:20”） |
| `LocateVex` | 根据城市名称查找城市在图中的下标 |
| `AddVertex` | 添加城市顶点（限制坐标在地图区域内） |
| `AddArc` | 添加城市间交通边（飞机/火车） |
| `AddSchedule` | 为交通边添加班次信息 |
| `DeleteVertexByName` | 删除指定城市（自动清理关联边，修正节点索引） |

### 2. 核心算法函数
| 函数名 | 功能 |
|--------|------|
| `SolveSingleLeg` | 求解两个城市间的单段最优路线（指定交通方式、规划策略） |
| `SolveWithVias` | 求解含途经城市的完整最优路线（排列组合途经顺序，遍历所有可能） |

## 编译运行说明
### 环境依赖
- 编译器：支持MinGW-w64（UCRT）或MSVC；
- 图形库：EasyX图形库（需提前安装）；
- 字符集：UTF-8（代码中指定`#pragma execution_character_set("utf-8")`）。

### 编译注意事项
- MinGW-w64（UCRT）环境下，代码内置`__iob_func`兼容补丁，解决EasyX依赖的旧版MSVCRT符号问题；
- 需禁用CRT安全警告（`_CRT_SECURE_NO_WARNINGS`）；
- 启用UNICODE宏，支持中文城市名称处理。

### 运行流程
1. 初始化：调用`InjectTestData()`加载内置的城市、交通边、班次数据；
2. 路线规划：调用`SolveSingleLeg`（无途经）或`SolveWithVias`（含途经），指定起点、终点、出发时间、规划策略（1=最短时间，2=最低成本）、交通类型（1=飞机，2=火车）；
3. 结果输出：解析`GlobalResult`结果，获取完整路线、班次、总耗时、总成本等信息，通过图形界面展示。

## 扩展说明
1. **新增城市**：调用`AddVertex`添加城市（名称、三字码、地图坐标），再通过`AddArc`添加与其他城市的交通边，最后用`AddSchedule`添加班次；
2. **删除城市**：调用`DeleteVertexByName`，自动清理该城市的所有入边/出边，无需手动维护索引；
3. **自定义策略**：修改`SolveSingleLeg`中`decision`分支逻辑，可扩展更多规划策略（如优先少换乘、优先高铁等）；
4. **地图扩展**：调整`SCREEN_WIDTH`/`SCREEN_HEIGHT`/`SIDEBAR_WIDTH`参数，可适配不同尺寸的地图界面。

## 注意事项
1. 城市坐标限制在地图区域内（左右40像素，上下40像素），超出会自动修正；
2. 换乘等待时间：飞机默认≥120分钟，火车默认≥60分钟，可修改`SolveSingleLeg`中`min_wait`变量调整；
3. 跨天时间计算：以1440分钟（24小时）为一天，输出自动标注天数；
4. 内存管理：删除城市/边时已做基础内存释放，批量操作后建议检查内存泄漏（可扩展`FreeScheduleList`函数释放班次链表）。
