## 交通咨询系统 (Traffic Consulting System) - 技术说明文档

### 1. 系统功能概述
本系统是一个基于 C++ 和 EasyX 图形库开发的交互式交通调度平台。
它利用邻接表存储城市与交通线路数据，通过 Dijkstra 算法的变体实现多约束下的最优路径规划（最快时间或最低成本），并提供图形化的地图界面进行交互。

### 2. 核心数据结构
系统采用图的**邻接表（Adjacency List）**存储方式，优化空间利用率：
- `VNode` (顶点结构)：存储城市名称、三字代码及地图坐标 (x, y)。
- `ArcNode` (边结构)：存储与相邻城市的关系（里程、交通方式）。
- `Schedule` (班次链表)：挂载在边上的详细班次信息，包含出发/到达时间及票价。
- `PathState` (状态结构)：算法运行时的中间状态，用于记录累积的最优值、前驱节点及选定班次。

### 3. 算法实现原理
本系统核心为适配交通网络特性的路径寻优算法：
1. **多维度松弛条件**
    - 最快到达：以到达时间绝对值作为松弛条件，最小化总耗时（含换乘）。
    - 最省钱：以累计费用为主权重，时间作为辅助优先级。
2. **中转等待逻辑**
    计算时自动识别中转站点，强制叠加基础候车时间：飞机 120 分钟 / 火车 60 分钟。
3. **跨天计算**
    通过 `day_offset` 处理跨天凌晨班次，保证 24 小时循环时刻表计算准确。
4. **途径点规划**
    使用 `std::next_permutation` 对用户指定途径点做全排列遍历，求解带途经约束的最优路径组合。

### 4. 关键交互逻辑
- **图形渲染**：使用 `BeginBatchDraw()` 双缓冲绘图，消除界面刷新闪烁，地图动画流畅。
- **事件驱动**：轮询 `peekmessage` 获取鼠标输入，坐标映射至按钮/城市节点，实现点击交互。
- **动态分辨率**：运行时可修改窗口分辨率，调用 `initgraph` 重绘画布，自动适配地图坐标边界。

### 5. 数据与扩展性
- **数据注入**：内置 `InjectTestData()` 内存初始化函数，预置全国主要交通枢纽航班、列车数据。
- **节点维护**：支持实时增删城市与交通线路；`DeleteVertexByName` 自动清理链表指针、重构邻接表，规避内存泄漏与野指针问题。

## 如何开始
### 环境要求
需提前安装 EasyX Graphics Library

### 编译设置
编译器需支持 Unicode 编码与 C++11 标准

### 操作指南
1. **路径查询**
在地图依次点击起点、终点（可选添加途径点），选择交通工具与优化目标，点击「开始最优检索」。
2. **线路管理**
点击「管理节点与联动」打开配置向导，自定义新增航线、列车班次。
3. **窗口缩放**
点击「切换分辨率」修改程序窗口尺寸。

## 开发提示
程序内置 UCRT MinGW 环境兼容补丁，修复旧版 MSVCRT `__iob_func` 链接报错，可在新版 Windows 环境正常编译运行。
