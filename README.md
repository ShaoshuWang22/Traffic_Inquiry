交通咨询系统 (Traffic Consulting System) - 技术说明文档
1. 系统功能概述
本系统是一个基于 C++ 和 EasyX 图形库开发的交互式交通调度平台。它利用邻接表存储城市与交通线路数据，通过 Dijkstra 算法的变体实现多约束下的最优路径规划（最快时间或最低成本），并提供图形化的地图界面进行交互。
2. 核心数据结构
系统采用图的**邻接表（Adjacency List）**存储方式，以优化空间利用率：
VNode (顶点结构)：存储城市名称、三字代码及地图坐标 (x, y)。
ArcNode (边结构)：存储与相邻城市的关系（里程、交通方式）。
Schedule (班次链表)：挂载在边上的详细班次信息，包含出发/到达时间及票价。
PathState (状态结构)：算法运行时的中间状态，用于记录累积的最优值、前驱节点及选定班次。
3. 算法实现原理
本系统的核心在于针对“交通网”特性的路径寻优：
多维度松弛条件：
最快到达：以到达时间的绝对值作为松弛条件，确保总耗时（含换乘）最小。
最省钱：以累计费用作为主权重，时间作为辅助优先级。
中转等待逻辑：算法在计算时，动态判断是否为中转站点，并强制加入 min_wait（飞机 120 分钟/火车 60 分钟）的候车时间。
跨天计算：通过 day_offset 逻辑，处理跨天凌晨出发或到达的班次，确保时刻表在 24 小时循环下的准确性。
途径点规划：利用 std::next_permutation 对用户选定的途径点进行全排列遍历，求解带约束的最优路径组合。
4. 关键交互逻辑
图形渲染：采用 BeginBatchDraw() 实现双缓冲绘图，消除了界面刷新时的闪烁，确保地图动画流畅。
事件驱动：通过轮询 peekmessage 获取鼠标输入，将坐标映射至 UI 按钮区域或地图城市节点，实现“点击即交互”。
动态分辨率：系统支持运行时修改分辨率，通过 initgraph 重新初始化画布，并自动调整地图坐标边界，适配不同屏幕需求。
5. 数据与扩展性
数据注入：程序采用 InjectTestData() 进行内存态数据初始化，包含全国主要交通枢纽的航班与列车信息。
节点维护：支持实时添加与删除城市及关联线路。系统在执行 DeleteVertexByName 时，会自动清理链路指针并重构邻接表，防止内存泄漏和野指针引用。
如何开始
环境要求：确保安装 EasyX Graphics Library。
编译设置：使用支持 Unicode 和 C++11 的编译器。
操作指南：
查询：在地图上依次点击起点、终点（可选添加途径点），选择交通工具与决策目标，点击“开始最优检索”。
管理：通过“管理节点与联动”按钮开启向导，可自定义增加航线或列车班次。
缩放：点击“切换分辨率”按钮，可改变系统工作窗口尺寸。
开发提示
本程序已内置针对 UCRT MinGW 环境的兼容补丁，解决了旧版 MSVCRT 库函数 __iob_func 的链接问题，确保在现代 Windows 环境下编译无误。
