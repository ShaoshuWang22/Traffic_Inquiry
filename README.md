# 1. 交通网络图结构设计与增删查改算法

## 1.1 图结构设计

交通咨询系统本质上是一个带权有向图问题。

其中：

* 城市表示顶点；
* 城市之间的交通线路表示边；
* 航班和列车班次作为边上的附加信息。

系统采用**邻接表存储结构**。

主要数据结构如下：

---

## （1）Schedule 班次结构体

```cpp
typedef struct Schedule {

    TCHAR no[20];

    int dep_time;

    int arr_time;

    double cost;

    Schedule* next;

}Schedule;
```

用于保存某条交通线路上的具体班次信息。

主要字段：

| 字段       | 作用       |
| -------- | -------- |
| no       | 班次编号     |
| dep_time | 出发时间     |
| arr_time | 到达时间     |
| cost     | 费用       |
| next     | 多个班次链表连接 |

例如：

北京 → 上海：

可能存在：

```
CA1201
08:00 - 10:00
500元

CA1302
15:00 - 17:00
400元
```

因此一条线路对应多个 `Schedule` 节点。

---

## （2）ArcNode 边节点结构体

```cpp
typedef struct ArcNode{

    int adjvex;

    int distance;

    int type;

    Schedule* sch_list;

    ArcNode* nextarc;

}ArcNode;
```

用于表示城市之间的交通关系。

其中：

* `adjvex`

表示目标城市编号。

* `distance`

表示线路距离。

* `type`

表示交通类型：

```
1  飞机

2  火车
```

* `sch_list`

连接该线路上的所有班次。

---

## （3）VNode 城市节点结构体

```cpp
typedef struct VNode{

    TCHAR name[30];

    TCHAR code[10];

    int x,y;

    ArcNode* firstarc;

}VNode;
```

表示地图中的城市。

其中：

* `name`

城市名称。

* `x,y`

EasyX地图坐标。

* `firstarc`

连接该城市出去的所有线路。

---

## （4）Graph 图结构

```cpp
typedef struct{

    AdjList vertices;

    int vexnum;

}Graph;
```

保存整个交通网络。

因此整个系统的数据关系为：

```
Graph

 |

VNode(城市)

 |

ArcNode(交通线路)

 |

Schedule(班次)
```

这种设计既满足图算法需求，也方便地图显示。最终版代码中正是采用这种“城市节点 + 邻接边 + 班次链表”的结构。

# 2. 城市和线路增删查改算法设计

交通咨询系统不仅需要查询路线，还需要支持交通网络维护。

主要函数：

```
LocateVex()

AddVertex()

AddArc()

AddSchedule()

DeleteVertexByName()

DeleteNode()

FreeArcList()

FreeScheduleList()
```

---

# 2.1 城市查找算法 LocateVex()

功能：

根据城市名称查找城市编号。

算法流程：

```
输入城市名称

↓

遍历vertices数组

↓

比较name字段

↓

找到返回下标

↓

否则返回不存在
```

例如：

查询：

```
北京
```

遍历：

```
0 上海

1 北京 √

2 广州
```

返回：

```
1
```

该函数是所有图操作的基础。

例如：

添加线路：

必须先通过：

```
LocateVex()
```

找到：

```
起点编号

终点编号
```

才能建立边。

---

# 2.2 城市增加 AddVertex()

功能：

向图中增加新的城市节点。

实现步骤：

1. 检查城市是否存在：

```
LocateVex()
```

2. 创建新的：

```
VNode
```

3. 保存：

* 城市名称
* 三字码
* 地图坐标

4. 初始化：

```
firstarc=NULL
```

5. 增加：

```
G.vexnum++
```

---

# 2.3 线路增加 AddArc()

功能：

建立两个城市之间的交通连接。

例如：

```
北京 → 上海
```

执行：

1. 查找：

```
北京编号

上海编号
```

2. 创建：

```
ArcNode
```

3. 设置：

```
adjvex=上海编号

type=飞机/火车

distance=距离
```

4. 插入邻接表。

---

# 2.4 班次增加 AddSchedule()

由于同一线路可能有多个班次，因此采用链表。

例如：

```
北京

 |

上海

 |

Schedule1

Schedule2

Schedule3
```

添加时：

创建新的：

```
Schedule
```

并连接：

```
sch_list
```

这样路径算法可以遍历所有可能班次。

---

# 2.5 删除算法

删除城市时不能只删除节点。

因为城市可能被多个线路连接。

因此需要：

1. 删除该城市所有出边：

```
FreeArcList()
```

2. 删除所有班次：

```
FreeScheduleList()
```

3. 删除其他城市指向该城市的边。

4. 更新：

```
adjvex编号
```

保证图结构连续。

# 3. 时间处理算法设计

交通规划区别于普通最短路径问题，因为边权不是固定值。

例如：

```
北京→上海
```

可能：

飞机：

```
8:00出发
10:00到达
```

火车：

```
9:00出发
18:00到达
```

因此需要时间计算。

主要函数：

```
TimeToMin()

MinToTimeStr()
```

---

## 3.1 TimeToMin()

作用：

将：

```
08:30
```

转换为：

```
510分钟
```

算法：

读取：

```
小时h

分钟m
```

计算：

```
h*60+m
```

方便：

* 比较时间先后；
* 计算等待时间；
* 处理跨天情况。

---

## 3.2 MinToTimeStr()

作用：

将：

```
510
```

转换：

```
08:30
```

用于最终界面显示。

# 4. 最快到达路径算法设计

核心函数：

```
SolveSingleLeg()
```

目标：

给定：

```
起点

终点

出发时间
```

寻找：

总耗时最小路线。

---

## 4.1 算法思想

普通Dijkstra算法：

边权：

```
距离
```

但是交通系统：

边权动态变化：

```
等待时间
+
乘车时间
+
换乘时间
```

因此系统采用：

**基于状态更新的最短时间搜索算法。**

状态结构：

```cpp
PathState
{
    best_value;

    pre_vex;

    chosen_sch;

    arrival_time;
}
```

保存：

* 当前最优时间；
* 前驱城市；
* 选择班次；
* 到达时间。

---

## 4.2 算法流程

初始化：

```
起点best_value=0

其他城市=INF
```

然后：

不断扩展当前最优城市。

对于当前城市：

遍历：

```
ArcNode
```

对于每条线路：

遍历：

```
Schedule
```

计算：

```
等待时间

+
旅行时间

+
换乘时间
```

得到：

```
新的到达时间
```

如果：

```
new_time < best_value
```

则更新。

---

最终：

得到：

```
GlobalResult
```

保存：

* 路径；
* 班次；
* 时间；
* 费用。

---

# 5. 最省费用路径算法设计

与最快路线类似。

区别：

优化目标不同。

最快模式：

目标：

```
min(总时间)
```

省钱模式：

目标：

```
min(总费用)
```

算法仍然使用：

```
SolveSingleLeg()
```

通过改变：

```
best_value
```

比较方式实现。

例如：

最快：

```cpp
if(new_time < best)
```

最省钱：

```cpp
if(new_cost < best)
```

因此同一个搜索框架支持多个优化目标。

# 6. 多途经点路径规划算法

函数：

```
SolveWithVias()
```

作用：

支持：

```
北京

↓

上海

↓

```

这种路线。

---

算法：

将复杂问题拆分：

```
北京→上海

上海→广州
```

分别调用：

```
SolveSingleLeg()
```

得到：

每一段最优路线。

然后：

合并：

```
full_path
```

计算：

```
total_duration

total_cost
```

最终生成完整路线。
